# web-clipper 新機能提案 - 2026-04-09

## エグゼクティブサマリー

前回提案（2026-04-01）の AI 自動タグ付けとハイライト機能の実装が完了しました。2026 年の市場では Pocket/Omnivore の終了により代替ツールへの移行が加速しています。本提案では、ブラウザ拡張の開発と Markdown エクスポート、セマンティック検索を軸に 8 つの機能を提案します。

## 現状の機能セット

- 記事クリッピング: URL からメタデータと本文（Markdown）を自動抽出し保存する
- AI 要約生成: Cloudflare AI による記事要約（キャッシュ対応）
- AI 自動タグ付け: クリッピング時に AI が既存タグから最適なタグを自動提案する
- ハイライト: 記事本文内のテキスト選択による色分けハイライト保存とメモ付与に対応
- タグ管理: カスタムタグによる記事分類（多対多リレーション）
- 検索とフィルタ: タイトル/説明文の全文検索、ソース/タグ/既読状態によるフィルタ
- ソース自動分類: Twitter、Qiita、Zenn、GitHub など 11 種のソースに対応
- 認証: ユーザー名/パスワードと GitHub OAuth に対応。Cookie ベースのセッション管理
- ダーク/ライトモード: テーマ切り替えとローカルストレージ永続化
- View Transitions: ページ遷移アニメーション
- レスポンシブ UI: モバイル/タブレット/デスクトップ対応
- API ドキュメント: OpenAPI 3.0 仕様と Scalar UI による API リファレンス

## 競合・トレンド分析

### 市場の大変動: Pocket/Omnivoreの終了

2025〜2026 年にかけて Web クリッパー市場に大きな変動が発生しました。

- Omnivore は 2024 年に閉鎖した
- Pocket は 2025 年 5 月に Mozilla が終了を発表した。2025 年 11 月にユーザーデータを完全削除

この結果、Raindrop.io や Readwise Reader などの代替ツールへ移行するユーザーが増えています。サービス終了リスクへの懸念から、オープンソース・セルフホスト・データポータビリティが求められる傾向です。

### 主要競合の動向（2026年4月）

| ツール | 注目機能 | ユーザー規模 |
|--------|---------|-------------|
| Readwise Reader | AIチャットサイドバー、YouTube自動トランスクリプト、RSS統合 | 有料$7.99/月 |
| Obsidian Web Clipper | 自然言語データ抽出、サイト別テンプレート自動適用、ハイライト復元 | 30万DL |
| Web Highlights | Web/PDF/YouTube横断ハイライトとAI要約 | 20万DL |
| ClipAI | AI自動要約とスマートタグ、セマンティック検索、エクスポート | 4.8星 |
| Raindrop.io | 広告除去Article Parser、全文検索、アーカイブ機能 | 急成長中 |
| Stacks | AIリコール機能、コンテキスト付き保存 | 新興 |

### 2026年のトレンド

1. AI チャットインターフェースの標準化: 単発の要約を超え、保存コンテンツへの対話型 Q&A が標準になった
2. セマンティック検索: キーワード一致ではなく意味ベースの検索を ClipAI や Remio が提供
3. データポータビリティの必須化: Pocket 終了の教訓から、Markdown エクスポートが必須要件になった
4. ブラウザ拡張の不在は致命的: 全主要競合が Chrome/Firefox 拡張を提供している
5. マルチメディア統一管理: YouTube トランスクリプトや PDF を単一インボックスで管理する
6. Webhook/API による自動化: 保存イベントをトリガーにした外部サービス連携

## 新機能提案

### 🏆 優先度: High（すぐに着手すべき）

#### 提案1: ブラウザ拡張（Chrome Extension）

概要:
Chrome 拡張機能を開発し、閲覧中のページをワンクリックでクリッピングする。選択テキストの右クリックメニューからのハイライト保存にも対応する。

ユーザー価値:
ブラウザ拡張の不在は全競合に対する最大のギャップです。Pocket/Omnivore 終了で代替を探すユーザーにとって必須の機能です。前回から継続して最優先とします。

実装難易度は Medium です。

技術的アプローチは以下のとおりです。

- TypeScript + Chrome Extension Manifest V3 を採用する
- `packages/extension`として monorepo に追加する
- 既存の`POST /api/articles`エンドポイントをそのまま利用する
- `@web-clipper/shared`の Zod スキーマを拡張側でも再利用する
- Popup UI は Vue 3 で構築する
- Content Script でテキスト選択からハイライト保存を実現する
- 認証は API トークン方式を新規追加する（cross-origin 対応）
- Service Worker でオフライン時の保存キューイングに対応する

Obsidian Web Clipper と ClipAI Chrome Extension を参考にしています。

#### 提案2: エクスポート機能（Markdown / JSON / OPML）

概要:
保存記事・ハイライト・タグを Markdown/JSON/OPML 形式でエクスポートする。一括と個別の両方に対応する。

ユーザー価値:
Pocket/Omnivore 終了の教訓から、データポータビリティはユーザーの信頼を得る必須要件です。Obsidian Vault への直接エクスポートを提供し、ロックイン不安を解消します。

実装難易度は Low です。

技術的アプローチは以下のとおりです。

- `GET /api/articles/export`エンドポイントを追加する
- format クエリパラメータで markdown/json/opml を切り替える
- 記事本文は既に Markdown で保存済みのため、フロントマターを付加して出力する
- ハイライトは Markdown 内に blockquote 形式で埋め込む
- JSON 形式では記事・ハイライト・タグの完全なリレーション情報を含める
- 一括エクスポートは ZIP ファイルで返却する（1 記事 1 ファイル）
- アプリケーション層に`ExportArticlesCommand`を追加する

ClipAI JSON/Markdown Export と Obsidian Markdown Format を参考にしています。

### 📋 優先度: Medium（次のマイルストーンで検討）

#### 提案3: セマンティック検索

概要:
現在のキーワード一致検索に加え、自然言語による意味ベースの検索を実現する。曖昧なクエリでも関連記事を返せるようにする。

ユーザー価値:
記事が蓄積するほど「あの記事なんだっけ」問題が深刻になる。キーワードを正確に覚えていなくても関連記事を発見できるようにする。

実装難易度は Medium です。

技術的アプローチは以下のとおりです。

- Cloudflare AI の Embedding モデルでテキスト埋め込みベクトルを生成する
- Cloudflare Vectorize にインデックスを保存する
- `clipArticle`コマンド実行時にベクトルを生成・保存する
- 検索クエリもベクトル化し、コサイン類似度でランキングする
- 既存のキーワード検索との組み合わせでハイブリッド検索を実装する
- `ArticleQueryService`にセマンティック検索メソッドを追加する

ClipAI Semantic Search（Pro 版）と Cloudflare Vectorize を参考にしています。

#### 提案4: AIチャットアシスタント

概要:
保存した記事に対してチャット形式で質問できる機能を追加する。主要な論点の整理や著者の結論の要約を対話的に行える。

ユーザー価値:
Readwise Reader の Ghostreader 統合がチャットサイドバーに昇格した。対話型 AI が 2026 年の標準機能となっている。単発の AI 要約を超えた深い理解を支援する。

実装難易度は Medium です。

技術的アプローチは以下のとおりです。

- `POST /api/articles/:id/chat`エンドポイントを追加する
- 記事本文をシステムプロンプトのコンテキストとして渡す
- Cloudflare AI の LLM でストリーミングレスポンスを返す
- ドメイン層に`ArticleAssistant`ドメインサービスインターフェースを追加する
- フロントエンドにチャットサイドバーUI を実装する
- チャット履歴は D1 に保存し、次回アクセス時に復元する

Readwise Reader Ghostreader Chat と Stacks AI Assistant を参考にしています。

#### 提案5: RSSフィード購読

概要:
RSS フィード URL を登録し、新着記事を自動でクリッピングする。Cloudflare Workers の Cron Trigger で定期実行する。

ユーザー価値:
お気に入りのブログやメディアの新着記事を手動保存する手間を省けます。Readwise Reader が統合済みで好評な機能です。

実装難易度は Medium です。

技術的アプローチは以下のとおりです。

- D1 に`feeds`テーブルと`feed_entries`テーブルを追加する
- Cron Trigger で 15 分間隔の定期実行を設定する
- 既存の`clipArticle`コマンドを再利用してフィードエントリをクリッピングする
- ドメイン層に`Feed`集約を新設する
- Web UI にフィード管理画面を実装する

Readwise Reader RSS Integration と Raindrop.io Feed Support を参考にしています。

### 設計上の注意点

ブラウザ拡張の認証について:
Cookie ベースセッションは cross-origin で動作しません。API トークン方式の追加が必要です。`ApiToken`はユーザー集約の関連エンティティとして設計します。

セマンティック検索について:
Vectorize はインフラ層の実装詳細です。ドメイン層では「記事の類似度検索」という概念のみ定義します。具体的なベクトル DB への依存を避けます。

Feed 集約の境界について:
`Feed`と`Article`は別集約です。フィードエントリの取得から`clipArticle`コマンド発行の流れで結果整合性を保ちます。

Webhook とドメインイベントについて:
将来的にドメインイベントパターンを導入する。`ArticleClipped`等のイベントを発行し、Webhook はサブスクライバーとして実装する。
