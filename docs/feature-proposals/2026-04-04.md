# web-clipper 新機能提案 - 2026-04-04

## エグゼクティブサマリー

AI Auto-Tagging の実装完了を受け、次の成長フェーズとしてブラウザ拡張機能と AI チャットアシスタントを提案します。
2026 年の競合トレンドは「保存→ワークフロー自動化」への進化です。
Cloudflare Workers AI + D1 の強みを活かし、クリップ後の知識活用体験を強化します。

## 現状の機能セット

- 記事クリッピング: URL からメタデータ（OGP 画像、タイトル、説明文）を自動取得し保存する
- AI 要約生成: Cloudflare Workers AI（llama-3.1-8b-instruct）による自動要約
- AI 自動タグ付け: 記事コンテンツに基づくタグ提案・自動適用（2026-04-03 実装完了）
- タグ管理: ユーザー定義タグ、多対多関連、件数表示、検索可能ドロップダウン
- 検索・フィルタ: タイトル/説明文の全文検索、タグフィルタ、ソースフィルタ、既読フィルタに対応
- ソース自動分類: Twitter、Qiita、Zenn、GitHub、Dev.to、Medium 等 11 種に対応
- 認証: ユーザー名/パスワード + GitHub OAuth、Cookie ベースセッション管理
- PWA 対応: Web Share Target API による OS 共有連携
- ダーク/ライトモード: テーマ切り替えと永続化
- View Transitions: ページ遷移アニメーション
- API ドキュメント: OpenAPI 仕様 + Scalar UI による API リファレンス

## 競合・トレンド分析

### 主要競合の動向（2026 年）

| ツール | 注目機能 | web-clipper との差分 |
|--------|---------|-------------------|
| Obsidian Web Clipper | テンプレートシステム、AI Interpreter、ローカルファースト、30 万 DL | ブラウザ拡張、テンプレート、ローカル保存 |
| Readwise Reader | Ghostreader チャット（GPT-5 対応）、音声 Daily Review、Webhook 連携 | ドキュメント QA、音声化、外部連携 |
| PixieBrix | ローコード自動化 Mod、AI エンリッチメント、CRM/Sheets 連携 | ワークフロー自動化、外部サービス連携 |
| Thunderbit | 自然言語での構造化データ抽出、マルチ AI エージェント | 構造化抽出、バッチ処理 |
| ClipAI | セマンティック検索（自然言語クエリ） | 意味ベース検索 |

### 2026 年の主要トレンド

1. 保存からワークフロー自動化へ: クリップ後に Webhook や CRM、Sheets 等へ自動連携する流れが加速している
2. 構造化データ抽出: 生 HTML ではなくプロパティやタグとしてクエリ可能な形式で保存する傾向にある
3. ドキュメント対話型 AI: 保存したコンテンツに対するチャット QA が標準化しつつある
4. マルチフォーマット対応: PDF、YouTube、RSS、ニュースレターなどを一元管理する方向に進化している
5. セマンティック検索: キーワード一致ではなく、意味ベースの検索機能が求められている
6. プライバシーとローカルファースト: ベンダーロックインのないデータ所有権が重視されている

## 新機能提案

### 優先度: High（すぐに着手すべき）

#### 提案 1: ブラウザ拡張機能（Chrome / Firefox）

- 概要: ワンクリックで閲覧中のページを保存するブラウザ拡張である。コンテキストメニューやポップアップ UI でタグ選択・メモ追加にも対応する
- ユーザー価値: 現在は web-clipper を開いて URL を手入力する必要がある。ブラウザ拡張により保存のフリクションが劇的に低下する。Obsidian Web Clipper は 30 万 DL を達成しており、ブラウザ拡張は Web クリッパーの基本的な UX である
- 実装難易度: Medium
- 技術的アプローチ:
  - Chrome Extension Manifest V3 + TypeScript で実装する
  - monorepo に `apps/extension/` を追加し pnpm workspace に統合する
  - `packages/shared` の Zod スキーマを共用し API 契約を統一する
  - Background Service Worker から `POST /api/articles` を呼び出す
  - Popup UI は軽量な Preact または vanilla TS で構築する
  - 認証は Cookie ベースセッションまたは API トークン方式を検討する
  - `content_scripts` で Selection API を利用し選択テキストのクリップにも対応可能である
- 参考: Obsidian Web Clipper、Notion Web Clipper

#### 提案 2: AI チャットアシスタント（ドキュメント QA）

- 概要: 保存済み記事に対して自然言語で質問できるチャット機能である。「この記事の主張の根拠は何か」等の QA が可能になる
- ユーザー価値: Readwise Reader の Ghostreader がチャット UI に進化し高評価を得ている。保存した記事を「読む」だけでなく「対話する」体験は知識の定着を促進する
- 実装難易度: Medium
- 技術的アプローチ:
  - 記事詳細ページにチャットパネルを Vue コンポーネントとして追加する
  - `POST /api/articles/:id/chat` エンドポイントを新設する
  - Cloudflare Workers AI に記事コンテンツとユーザー質問を送信する
  - 既存の `ContentExtractor` で取得した Markdown をコンテキストとして活用する
  - ドメイン層に `ChatMessage` VO と `ArticleChat` ドメインサービスを新設する
  - チャット履歴は D1 の `article_chats` テーブルに保存する
  - Workers AI の streaming 対応でリアルタイムレスポンスを実現する
- 参考: Readwise Ghostreader

### 優先度: Medium（次のマイルストーンで検討）

#### 提案 3: フルテキストコンテンツ検索

- 概要: 記事の本文を D1 に保存し SQLite FTS5 による全文検索を実現する。現在はタイトルと説明文のみ検索可能である
- ユーザー価値:「あの記事で読んだフレーズ」で検索できる体験を提供する。ClipAI のセマンティック検索に対抗できる
- 実装難易度: Medium
- 技術的アプローチ:
  - `articles` テーブルに `content` カラムを Markdown 形式で追加する
  - D1 の FTS5 仮想テーブルでインデックスを構築する
  - `ArticleRepository` に FTS 検索オプションを追加する
  - 将来は Cloudflare Vectorize でセマンティック検索にも拡張可能である

#### 提案 4: RSS フィード購読

- 概要: RSS フィードを登録し新規エントリを自動クリップする機能である。Cloudflare Workers の Cron Trigger で定期実行する
- ユーザー価値: ブログやニュースサイトの更新を自動収集できる。Readwise Reader の RSS 機能に対抗する
- 実装難易度: Medium
- 技術的アプローチ:
  - `feeds` テーブルを新設し `FeedSubscription` エンティティをドメイン層に追加する
  - Cron Trigger の `scheduled` ハンドラで定期フェッチし `clipArticle` を呼び出す
  - RSS パースには軽量な `fast-xml-parser` を使用する

#### 提案 5: ハイライト・アノテーション

- 概要: 記事内のテキストを選択してハイライトしメモを付けられる機能である
- ユーザー価値: Readwise Reader や Obsidian の主要機能である。記事を能動的に読む体験を促進する
- 実装難易度: High
- 技術的アプローチ:
  - 記事本文の HTML レンダリング上で Selection API を使用する
  - `highlights` テーブルを新設し `Highlight` VO をドメイン層に追加する
  - テキスト選択時にポップオーバー UI でハイライト色選択やメモ入力を提供する

### 優先度: Low（長期的に検討）

#### 提案 6: マルチフォーマット対応（PDF / YouTube）

- PDF テキスト抽出や YouTube トランスクリプト取得・保存に対応する
- ソース分類に PDF と YouTube を追加する

#### 提案 7: エクスポート・外部連携

- Obsidian Vault 同期や Notion API 連携を実現する
- JSON / CSV / Markdown 一括エクスポートを提供する
- Webhook 通知で Readwise 型のイベント駆動連携を可能にする

#### 提案 8: 音声化・Daily Review

- 保存記事の AI 要約を Cloudflare Workers AI TTS で音声化する
- ランダムに過去記事を提示する Daily Review 機能で復習を促進する

## DDD / クリーンアーキテクチャ観点のメモ

### ブラウザ拡張機能（提案 1）

レイヤー影響はプレゼンテーション層のみです。
新しいクライアントが既存 API を呼び出す構成であり、ドメインモデルへの影響はありません。
`packages/shared` のスキーマを共用することで API 契約を統一できます。
API トークン認証を追加する場合は `ApiToken` エンティティの新設が必要です。

### AI チャットアシスタント（提案 2）

各レイヤーへの配置は以下のとおりです。

- ドメイン層: `ChatMessage` VO と `ArticleChat` ドメインサービスのインターフェース
- アプリケーション層: `chatWithArticle` コマンド
- インフラ層: `WorkersAiArticleChat` 実装
- プレゼンテーション層: `/api/articles/:id/chat` ルート

チャット履歴は Article 集約とは別の `ChatSession` 集約として設計します。
Article の肥大化を防ぐためです。

### フルテキスト検索（提案 3）

`Article` エンティティに `content` プロパティを追加します。
`ContentExtractor` が既にコンテンツ取得を担当しているため、保存の仕組みを追加するだけです。
`ArticleRepository` に FTS クエリメソッドを追加します。

### RSS フィード（提案 4）

`FeedSubscription` という新規集約を導入します。
`NewEntryDiscovered` ドメインイベントで `clipArticle` コマンドをトリガーします。
フィード管理は Article 管理とは別のコンテキストですが、現在の規模では同一モジュール内で十分です。
